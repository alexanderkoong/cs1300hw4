Chong Qing House Responsive Redesign

A case study in improving usability and accessibility for one of my favorite restaurant's online website

Background/Motivation

Chong Qing House is one of my favorite restaurants in Providence, RI. While their food is excellent, their online ordering website was always very hard to use. Additionally, it was not mobile-device friendly (webisite would not resize), so I couldn't order when I wasn't by my computer. This project demonstrates my process of analyzing, redesigning, and implementing a more user-friendly and responsive website that is personally meaningful to me.

The original Chong Qing House homepage viewed on a standard computer web browser.

Identifying Usability Problems

I evaluated the original website against three key usability criteria and performed an accessibility analysis:

Efficiency (6/10)

The original website contained numerous extraneous elements that hindered efficiency:

  • "Select Order Type" button is redundant; there is only one order type
  • Address change button is redundant; there is only one location
  • Logo with image of food takes up valuable space and hides cuts off some menu options
  • "Store Info" button offers no new infromation and instead creates a confusing loop in the navigation
  • DoorDash header also takes up valuable space and instead could be moved to "Select Order Type" button
  • Due to the inefficiencies in space, not all submenu options are easily viewable (Diet Dishes, Side, Order, and Dessert are all cut off)

Learnability (8/10)

While generally clean, there were initial barriers:

  • Unclear how to add items to cart (no "click to add" or cart icons)
  • Checkout button state (dimmed until cart has items) wasn't obvious

Memorability (8/10)

Once learned, the process was fairly intuitive, but users might forget:

  • How do I add a tip? Why does it say edit tip when there is no tip added? What if I don't want to tip?
  • That the checkout button requires items in cart to be clickable

Accessibility Issues (WebAIM WAVE Analysis)

  • Very low contrast between red text and yellow highlights in header
  • Keyboard navigation was extremely difficult - tip button was #223 in tab order
  • No ability to navigate between food subcategories using keyboard shortcuts
  • Not designed for all screen sizes; only designed for large computer screen
  • Default font resizing does not work; cannot translate website to different languages

Why This Matters/How This Impacts User Experience

These accessibility issues create significant barriers for many users:

  • Low contrast and ineffiencies in layout/spacing makes content very difficult to read for users with visual impairments
  • Poor keyboard navigation excludes users who cannot use a mouse
  • Lack of keyboard shortcuts prevents efficient navigation for assistive technology users
  • Non-responsive design makes the site unusable on mobile devices and tablets

Visual Redesign

Based on the problems identified, I decided to redesign the Chong Qing website. I first created a visual design style guide and mockups for various screen sizes.

New Design: Style Guide

The style guide features colors inspired by traditional Chinese aesthetics, with a rich red symbolizing luck and prosperity, complemented by a warm cream for balance. I aimed for a cleaner, sleeker look with improved contrast ratios and consistent typography.

Consistent Visual Cues

My redesign establishes a consistent visual language through carefully selected color combinations and component styling that remains uniform across all screen sizes and interactive states.

  • Each interactive element maintains recognizable visual cues—such as the distinctive red border on selection, shadow effects on hover, and consistent iconography.
  • For actions like adding items to cart, my redesign ensures that users can easily identify and predict behavior regardless of device or context
  • This visual consistency extends to the structural elements of the interface, with containers, cards, and buttons sharing proportional spacing and styling relationships even as they adapt to different viewport sizes.
  • By maintaining these visual patterns throughout the experience, users can seamlessly transfer between desktop, tablet, and mobile interactions, reducing the learning curve associated with cross-device usage.

    Feedback/Responsive Implementation

    • After receiving feedback on my initial designs, I realized there wasn't enough differentiation between layouts at different screen sizes.
    • The text on phone-sized/tablet screens would be too small to read with my original approach.
    • I refined my responsive strategy to better accommodate each device type:
      • A three-column layout for large computers
      • A two-column layout for tablets (with the "Your Order" section relocated to the bottom)
      • A single-column layout for phones that allows users to comfortably scroll through all content while maintaining readability

    Desktop

    Three-column layout with ample space for menu items, allowing users to browse efficiently with categories on the left, menu items in the center, and cart on the right.

    Tablet

    Adapted layout with appropriate scaling, collapsing to a two-column design that maintains usability while optimizing for medium-sized screens.

    Mobile

    Single-column layout focusing on essential content with an emphasis on scrolling to view the additional columns (ensuring text size remains large enough to read).

    How does this redesign impact usability?

    • Increased Efficiency: Removed redundant buttons and simplified navigation, which will allow users to more easily order food options. The updated website now shows all menu options at once and is overall simpler/cleaner - all while conveying the same information. The updated color scheme additionally makes it easier to read/identify menu options. Each menu option now additionally has an image associated with it.
    • Better Learnability/Intuitive Interactions: Added clear visual cues for adding items to cart and proceeding to checkout, eliminating confusion about how to add a menu item to cart. Also added in new buttons such as the +/- buttons to simplify adding/removing quantities of particular orders.
    • Interaction Resizing New implementation can be adapted to all screen sizes, allowing for people to view this webpage on mobile devices, tablets, and computers. This will allow people to easily order - regardless of device type.

      • Overall, my new design is sleeker, simpler/more intuitive to use - all while conveying the exact same information as the original Chong Qing website.

    How does this redesign impact accessibility?

    • Enhanced Color Contrast: Replaced the problematic red text on yellow background with higher contrast color combinations that meet WCAG standards, making content readable for users with visual impairments
    • Keyboard Navigation: Restructured the tab order to allow keyboard users to access important functions (like checkout) without navigating through hundreds of menu items first. This will allow disabled people who cannot use mouses to easily navigate the website.
    • Consistent Interactive Elements: Implemented clear visual indicators for clickable elements, visual patterns are now much more clear. This will help those with attention/learning/cognitive disabilities.
    • Proper Screen Resizing: Increased text, button, and title sizes to be usable regardless of screen size, making the site more usable for people with motor control disabilities.
    • Language/Font Sizing: Users can now easily translate the web page to a different language for clear ordering. Users who require larger default font sizes will now have a website that updates accordingly.

    Conclusion

    This project demonstrates my approach to identifying usability issues and implementing responsive design solutions. By focusing on user needs and following best practices in web design, I transformed a challenging interface into an intuitive and accessible experience.
